IEnumerable<User> CoreTweet.Rest.Users.Lookup ( params Expression< Func< string, object >>[]  parameters)
inline

Returns fully-hydrated user objects for up to 100 users per request, as specified by comma-separated values passed to the user_id and/or screen_name parameters.

This method is especially useful when used in conjunction with collections of user IDs returned from GET friends/ids and GET followers/ids.

GET users/show is used to retrieve a single user object.

Avaliable parameters:

string user_id (optional) : A comma separated list of user IDs, up to 100 are allowed in a single request. You are strongly encouraged to use a POST for larger requests.

string screen_name (optional) : A comma separated list of screen names, up to 100 are allowed in a single request. You are strongly encouraged to use a POST for larger (up to 100 screen names) requests.

bool include_entities (optional) : The entities node that may appear within embedded statuses will be disincluded when set to false.

Returns the size of the specified user's profile banner. If the user has not uploaded a profile banner, a HTTP 404 will be served instead. This method can be used instead of string manipulation on the profile_banner_url returned in user objects as described in User Profile Images and Banners.

Provides a simple, relevance-based search interface to public user accounts on Twitter. Try querying by topical interest, full name, company name, location, or other criteria. Exact match searches are not supported.

Only the first 1,000 matching results are available.

string q (required) : The search query to run against people search.

int count (optional) : The number of potential user results to retrieve per page. This value has a maximum of 20.

inline

Access the users in a given category of the Twitter suggested user list.

It is recommended that applications cache this data for no more than one hour.

string slug (required) : The short name of list or a category

string lang (optional) : Restricts the suggested categories to the requested language. The language must be specified by the appropriate two letter ISO 639-1 representation. Currently supported languages are provided by the GET help/languages API request. Unsupported language codes will receive English (en) results. If you use lang in this request, ensure you also include it when requesting the GET users/suggestions/:slug list.
