Detailed Description

Provides a set of methods for the wrapper of GET/POST lists/members.

Member Function Documentation

ListResponse CoreTweet.Rest.Members.Create ( params Expression< Func< string, object >>[]  parameters)
inline

Add a member to a list.

The authenticated user must own the list to be able to add members to it.

Note that lists can't have more than 500 members.

Note: Either a list_id or a slug is required.

If providing a list_slug, an owner_screen_name or owner_id is also required.

Available parameters:

- long list_id (required)

- string slug (required)

- long user_id (required)

- string screen_name (required)

- string owner_screen_name (optional)

- long owner_id (optional)

ListResponse CoreTweet.Rest.Members.CreateAll ( params Expression< Func< string, object >>[]  parameters)
inline

Adds multiple members to a list, by specifying a comma-separated list of member ids or screen names.

The authenticated user must own the list to be able to add members to it.

Note that lists can't have more than 500 members, and you are limited to adding up to 100 members to a list at a time with this method.

Please note that there can be issues with lists that rapidly remove and add memberships.

Take care when using these methods such that you are not too rapidly switching between removals and adds on the same list.

Note: Either a list_id or a slug is required.

If providing a list_slug, an owner_screen_name or owner_id is also required.

Available parameters:

- long list_id (required)

- string slug (required)

- string / IEnumerable<long> user_id (optional)

- string / IEnumerable<string> screen_name (optional)

- string owner_screen_name (optional)

- long owner_id (optional)
